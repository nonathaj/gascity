Gas City is an orchestration-builder SDK for multi-agent systems. This docs
tree is organized for engineers who want to run Gas City: understand how the
system fits together, install the toolchain, run a local city, and then find
the relevant subsystem.

If you are new, start with the [six primitives](/concepts/primitives) — the
shortest path to the mental model. Gas City is built from six primitives: an
**agent** is *who* does the work (sessions, providers, and pools are derived
under it), a **bead** is *what* the work is, a **formula** is *how* the work is
carried out, a **rig** is *where* it happens (a project registered with the
city), a **pack** is what *configures* the system — the City is the local
(root) pack; it imports shared packs — and an **event** is fired by activity so
you can *observe* what happened. The
[Architecture Overview](/concepts/architecture-overview) then shows how these
fit together and how work flows through them, with diagrams.

## Repository Context

Gas City is the open-source SDK extracted from Gas Town. The public docs now
separate guidance for running Gas City from historical planning material so new
readers can get oriented without reading every audit and roadmap first.
